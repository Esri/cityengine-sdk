Asset and texture search: Now the search for an inserted asset (or texture) is always relative to the rule file (this is also the case for imported rule files); the search order is as follows: 1) if userPath is absolute workspace path (start with "/") return that file ,2) try to resolve relative to "assets" in the project where the rule file resides 3) try to resolve relative to project. No search heuristcs are applied anymore, so for instance the asset "assets/windows/window1.obj" must be inserted as i("windows/window1.obj").
